GitHub 连击计划

Aug 1, 2016

GitHub 改版之后貌似就不显示 Longest streak 的时间了，不过用了Isometric Contributions这个 Chrome 插件还是可以看到的：

中间空白了一大段是因为生病住院。

虽说追求 GitHub 的连击没有什么意义，但是至少可以一定程度上反映是做了一些微小的工作的，另外必须给自己一些规则，要不然每天 update readme 也算 commit 就没什么意义了。

  1. 只有代码相关的 commit 才算做连击的内容；
  2. 不能够使用脚本修改之前的 commit；

然后看一下能够连击多少天吧。

另外，weekly-project这个 repo 貌似好久没有更新的，以前还打算每周完成一个 project 的，果然暑假待在家里全忘关了，现在也要把这个提上日程了。

少玩点守望先锋，多写点代码：）
