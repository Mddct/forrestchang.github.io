*args 与 **kwargs 的作用

当我们不确定向一个函数中传入多少参数时，或者想要传入列表或者元组形式的参数时，可以使用 *args 。当我们不确定向一个函数中传入多少个关键词数时，或者想要传入字典的值作为关键词参数，我们可以使用 ~*kwargs~。当然，args 与 kwargs 只是一种命名方式。

其中，在调用参数中含有 * 时， * 把序列或者元组解包（unpack）成位置参数， ** 将字典解包成关键词参数。

装饰器（Decorator）

装饰器本质上是一个 Python 函数，它可以让其他函数在不做任何变动的情况下增加额外的功能，它的返回值也是一个函数对象。

常用场景：插入日志、性能测试、事务处理、缓存、权限校验等。

垃圾回收机制（Garbage collection）

Python 中的垃圾回收是以引用计数为主，标记-清除和分代收集为辅。

引用计数
Python 在内存中存储每个对象的引用计数，如果计数变成0，该对象就会消失，分配给该对象的内存就会被释放。
标记-清除
一些容器对象，比如 list/dict/tuple/instance 等可能会出现引用循环，对于这些循环，垃圾回收器会定时回收这些循环。
分代收集
Python 把内存根据对象存货的时间划分为三代，对象创建之后，垃圾回收器会分配它们所属的代。每个对象都会被分配一个代，而被分配更年轻的代是被优先处理的，因此越晚创建的对象越容易被回收。

多线程（Multi-threading）

Python 并不支持真正意义上的多线程。Python 中有一个叫 Global Interpreter Lock (GIL) 的东西，它能确保你的代码中永远只有一个线程在执行。经过 GIL 的处理会增加执行的开销。这就意味着如果你先要提高代码的执行效率，使用 threading 不是一个明智的选择，当然如果你的代码是 IO 密集型，多线程可以明显提高效率，相反如果你的代码是 CPU 密集型，这种情况下多线程大部分是鸡肋。
