  • Scenario 1: The one-off model

    The problem if well defined by your boss. She gives you the data, which is small enough to look at and understand in MS Excel if you had to.

  • Scenario 2: The embedded model

    You and your team are collecting requirements form stakeholders on software project. There is a requirement for the user to be able to freehand draw shapes in the software, and for the software to figure out which shape it is and turn it into a crisp unambiguous version and label it appropriately.

    You quickly see that the best way to solve this problem is to devise and train a predictive model and embed it in your software product.

    What's key in this scenario is that you have the experience to notice that is best solved with a predictive model and the skills to devise, train and deploy it.

  • Scenario 3: The deep model

    You will be expected to build a deep understanding of one specific predictive model and use your experience and skill to improve and verify its accuracy as part of your routine duties.

Machine Learning 工程师的三重境界：

  1. 能够根据所给的数据做一些分析和预测。
  2. 能够根据具体的问题给出解决这个问题所需要使用什么模型。
  3. 能够对使用的模型有深入的了解，可以根据问题来优化和改进模型。

The Traditional Bottom-up Approach

A Better Approach

You need a top-down approach to machine learning. An approach where you focus on the actual result you want; working real machine learning problems from end-to-end using modern and "best of breed" tools and platforms.

Biggest Mistakes Developers Make and How To Avoid Them

  1. Not taking action.
  2. Picking problems that are too big.
  3. Implementing algorithms from scratch.
  4. Not sticking to a process.
  5. Not using resources.
