How do I prepare for the Google Summer of Code

https://www.quora.com/How-do-I-prepare-for-the-Google-Summer-of-Code-GSoC

  • Good understanding of one programming language of your choice
    • Learn basic syntax
    • Learn multiple major librarires and packages that are going to be used in any medium to large scale project (ex: Java Collections and C++ STL)
    • Look up the relevant libraries of your programming language and devote time in learning them.
  • Version Control
    • Learn Git (not GitHub)
    • Learn using GitHub
  • Learn to read the source code
    • [ ] Learn to Read the Source, Luke
    • How do you dive into large code bases?
  • Learn how to use Bug/Issue Trackers
    • Bug trackers are like the Version Control systems which we talked about earlier. In fact, most version control clients will have an implementation of the bug tracker themselves.
  • Submit Patches
    • Once you've learnt all of the above, you are ready to submit some patches, or bugfixes to a project of your choice. Choose any open source project which is written in a language you know, and more importantly, interests you, and go and browse through its bug tracker. Most bug trackers have issues marked as "volunteer" or "introductory". Search for such issues, or go with any issue which you think you can fix.
    • You should know:
      • What DVCS the project is using
      • What are their workflow for code contribution. (Most organisations have a defined way to submit a patch and you MUST follow that).
      • How to get involved with the community. Because even open source projects wont accept unauthorised/unreliable code from some newcomer.
  • Select an organisation/project and contribute
    • Browse through last year GSoC list of accepted organisation.
    • Once you've picked an organisation, start creating a good reputation for yourself.
      • Get in contact with the devs. Use IRC, subscribe to the developer Mailing List of that org
      • Submit patches. Nothing will help your case as much as your coding capability. Fix bugs. Even one is good. Start with the easistt of bugs. If in doubt, ask the devs to direct you to the bug they think will be good for you to start with.
      • Get familiar with their workflow. This includes, how they contribute code, their coding style, if they hold any online meetings(do attend) etc.
    • This should ideally be done at least 2-3 months in advance before the applications for GSoC opens next year.
  • Ask (Smart) Questions
    • Once you are up and running contributing to a project, you WILL face difficulties. You might not know how a particular piece of code is working, or you might not know about the required technology. This is the time for you to ASK. Ask the established developers in that project.
    • How to Ask Questions The Smart Way
  • Diversify and be active
    • It is generally considered a bad idea to apply to just one organisation. Even if you are a very active contributor to a specific project and are assured of getting selected, consider applying to another project. Go through the same cycle as you did with the first project.
  1. Pick an organisation that you like from the list of Orgs from 2015/2014/2013…
  2. Google around for the Organisations IRC channel.
  3. Connect to the IRC channel
  4. Say, "Hi, I'm new and I would like to contribute, can someone get me started?"
  5. Success. :D
